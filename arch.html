AI Lecture Note 架構導覽

更新：2026-06-03。此文件主要依據目前的 lib/ 與 functions/ 程式碼整理，arch.md 作為補充。目標閱讀時間：15 分鐘內。

閱讀方式

這個專案是 Flutter app 加 Firebase backend。前端採輕量 MVVM：畫面用 screens/ 與 widgets/ 組起來，狀態放在 viewmodels/，跨 SQLite、local files、Firebase 的流程集中在 repositories/ 與 services/。

1. 整體架構 2. 細部架構 3. lib 檔案導覽 4. 新增 Cloud Functions 5. 常用 trace 入口 6. 開發注意事項

1. 整體架構

App 入口是 lib/main.dart，先初始化 Firebase，再用 GoRouter 建三條主要路由。大部分 UI 狀態由 ChangeNotifier viewmodel 手動監聽，沒有引入 Provider 或 Riverpod。Cloud backend 目前在 functions/index.js，提供 chat 與 PDF 產生逐頁筆記兩個 HTTP Functions。

重點邊界：UI 不直接碰 Firebase Functions；Functions call 由 FirebaseFunctionClient 統一處理，資料庫 CRUD 則集中在 DatabaseHelper。

2. 細部架構

Navigation 架構

ShellRoute 會包住主要頁面並套上 LayoutWrapper。Lecture workspace 本身不顯示 app bar / bottom bar，所以 LayoutWrapper 會用 path 判斷是否在 /lecture/。

  • courseId 與 fileId 在 route 中是字串，但通常是 SQLite items.id。
  • Dashboard 點 folder/course 進 /course/:courseId；點非 folder 進 /lecture/:courseId/:fileId。
  • CourseDetails 可遞迴進 folder/course，也可進 lecture workspace。

Local SQLite 架構

SQLite singleton 是 DatabaseHelper.instance，資料庫檔名 lecture_system_v2.db，schema version 3。它同時管理檔案樹與聊天紀錄。

content 目前會放 recording 的完整 transcript，也可能放 AI note / notebook 內容。真正的 PDF、transcript segment、notes JSON/Markdown 多半存在 app documents，SQLite 只記路徑和索引資料。

Firebase / Storage / Firestore / Functions 架構

Flutter 端不直接呼叫 OpenAI。PDF 會先上傳到 Firebase Storage，再由 generateNotesFromPdf Function 下載、交給 OpenAI Responses API、把 notes JSON 寫回 Storage，並回傳 pages 或 notesStoragePath。

  • FirebaseFunctionClient 預設組出 https://<region>-<projectId>.cloudfunctions.net/<functionName>。
  • 可以用 FIREBASE_FUNCTIONS_BASE_URL 指到 emulator，也可以用 FIREBASE_CHAT_FUNCTION_URL / FIREBASE_NOTES_FUNCTION_URL 指定完整 URL。
  • generateNotesFromPdf 會寫 Firestore job state；目前環境已建立好即可。換 project 或重建 Firebase 環境時，要確認 Cloud Firestore database 已啟用，否則 Function 會因 Firestore NOT_FOUND 失敗。

4. 如果要新增 Cloud Functions 應該怎麼接

  1. 在 functions/index.js 新增 exports.yourFunction。沿用 handleCors(req, res)、requestPayload(req)、requiredString、sendError，回傳 plain JSON object。
  2. 在 lib/services/ 新增一個小型 service，使用 FirebaseFunctionClient.postJson。每個 Function 建議提供兩個 dart-define：FIREBASE_XXX_FUNCTION_NAME 與 FIREBASE_XXX_FUNCTION_URL。
  3. 如果流程需要 SQLite、local cache 或多個 services，放到 repositories/ 組合；UI state 再由 viewmodels/ 呼叫 repository。
  4. 如果 Function 會寫 Firestore 或讀寫 Storage，先確認 Firebase project、Storage bucket、Firestore database 都已建立，且 client 端的 firebase_options.dart 指到同一個 project。
  5. 部署前檢查 backend 語法與前端靜態分析：node --check functions/index.js、flutter analyze。本專案目前不把 flutter test 當作必要驗證步驟。

Backend 範本

exports.summarizeTranscript = functions
  .region(region)
  .runWith({ timeoutSeconds: 120, memory: "512MB" })
  .https.onRequest(async (req, res) => {
    if (handleCors(req, res)) return;

    try {
      const payload = requestPayload(req);
      const transcript = requiredString(payload, "transcript");

      // Do work here: OpenAI, Firestore, Storage, etc.
      res.json({ summary: transcript.slice(0, 200) });
    } catch (error) {
      sendError(res, error);
    }
  });

Flutter service 範本

import 'firebase_function_client.dart';

class TranscriptSummaryService {
  TranscriptSummaryService({FirebaseFunctionClient? functionClient})
    : _functionClient = functionClient ?? FirebaseFunctionClient();

  static const _functionName = String.fromEnvironment(
    'FIREBASE_TRANSCRIPT_SUMMARY_FUNCTION_NAME',
    defaultValue: 'summarizeTranscript',
  );
  static const _functionUrl = String.fromEnvironment(
    'FIREBASE_TRANSCRIPT_SUMMARY_FUNCTION_URL',
  );

  final FirebaseFunctionClient _functionClient;

  Future<String> summarize(String transcript) async {
    final response = await _functionClient.postJson(
      functionName: _functionName,
      overrideUrl: _functionUrl,
      body: {'transcript': transcript},
    );

    final payload = FirebaseFunctionClient.unwrapPayload(response);
    final summary = payload['summary'];
    if (summary is! String || summary.trim().isEmpty) {
      throw const FirebaseFunctionException('Missing summary.');
    }
    return summary.trim();
  }

  void dispose() => _functionClient.dispose();
}
Emulator 常用設定：--dart-define=FIREBASE_FUNCTIONS_BASE_URL=http://127.0.0.1:5001/<projectId>/us-central1。若只想覆蓋單一 Function，使用完整 URL dart-define 會更直接。

6. 開發注意事項

  • Firebase project 要對齊。目前 firebase_options.dart 的 web/android 是 ai-notes-555a6，iOS/macOS/windows 是 speech-recognition-7d363。跑桌面版時，很可能打到不同 Storage/Functions project。
  • Storage 有檔案不代表 notes 已完成。PDF source upload 成功後，還要等 Function 寫出 ai_note_jobs/{fileId}/notes/notes.json 或直接回傳 pages。
  • Firestore 是環境前置條件。generateNotesFromPdf 會寫 job state；目前若已經能看到 Firestore 資料庫畫面就不用再處理。只有換 project 或新建環境時，才需要先啟用 Cloud Firestore，否則會出現 gRPC NOT_FOUND。
  • SQLite 與 local files 是兩層。items.filePath 只記本機路徑；檔案本體在 app documents，例如 lecture_slides/、transcripts/、ai_notes/。
  • legacy 檔案不要當主流程。Azure STT、speech_to_text、static transcript data 目前沒有從 main.dart route 掛進 active app。
